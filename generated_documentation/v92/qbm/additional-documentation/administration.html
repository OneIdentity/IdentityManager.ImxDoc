Administration

In this section, you will find information about the administration portal and other components that are used for administrative purposes, like the authentication process.

1. The Administration portal

The main page is implemented in the DashboardComponent and contains this sub pages:

1.1. Overview

The main information about the portals, that are installed.

This is implemented by the StatusComponent and added using <imx-status></imx-status>.

1.2. Configuration

This contains all the configuration parameter, that can be changed on the server.

This is implemented by the ConfigComponent and added using <imx-config>.

1.3. Packages

An overview of all the packages, that are currently loaded.

This is implemented by the PackagesComponent and added using <imx-packages>.

1.4. Plugins

An overview of all the plugins, that are currently loaded by the server.

This is implemented by the PluginsComponent and added using <imx-plugins>.

1.5. Caches

An overview of all caches used by the server.

This is implemented by the CacheComponent and added using <imx-cache>.

1.6. Logs

An ongoing list of log messages, that are produces by the API server This is implemented by the LogsComponent and added using <imx-logs>.

1.7. API Documentation

A swagger generated API documentation.

This is implemented by the SwaggerComponent and added using <imx-swagger>.

2. Authentication

In this section you will find other component, that are used during authentication.

The login itself is handled by the AuthenticationService, while the frontend is build using the LoginComponent.
There is even a special route guard, for checking, if a user is logged in or not. An older way doing this is using the imx_SessionService.
