IMX Web - QBM Library

Listings

In this section you can find components, that are used to display large amounts of data (like the content of a filtered database table).

1. Data Source Toolbar

The DataSourceToolbarComponent is used to handle the interaction between the data source and the actual view.
It is defined in the DataSourceToolbarModule, together with the other sub components, that are used inside the toolbar.

The following features are available:

  • Filtering
  • Searching
  • Grouping
  • Ordering
  • Export and Configuration of the view

The definition of values and functions for these features should be defined using the DataSourceToolbarSettings interface.

2. Paginator

The DataSourcePaginatorComponent is used to handle the pagination of the data. This is done by updating the navigation state of its linked DataSourceToolbarComponent. The usage of a paginator is optional and for some scenarios it's not needed anyway.

3. View Components

There are several components, that are capable of viewing a large amount of data. They are defined in separate Angular modules and are as follows:

3.1. Data Table

The DataTableComponent ist the most used view component for our portals.

It renders an Angular Material Table with columns, that can be customized by using other sub components. These are defined in the DateTableModule, as well as the date table itself.

It is possible to display a auto generated view as well, which renders all columns that are defined in the data source as normal text columns.

3.2. Data Tiles

The DataTilesComponent can be used to render a tile view, which displays a DataTileComponent for each entry in the data source. Both components are part of the DataTileModule, but only the DataTilesComponent is exported.
All configurations has to be defined as inputs for the data tiles components.

3.3. Data Tree

The DataTreeComponent can be used to display hierarchical data. Other than the table, it uses a special data source, that can be defined by extending the abstract TreeDatabase class. The data tree is part of the DataTreeModule, together with other components and sub components, that can be used with the data tree like the DataTreeSearchResultsComponent. This component shows a non hierarchical view of an entry subset, because some parameter like a filter or a search string will flatten the result.

If you like to use a data tree with a data source toolbar functionality, there is a DataTreeWrapperComponent defined in the DataTreeWrapperModule. This component is combining a data tree with a data source toolbar, so it can be filtered and searched.

3.5. Other

There are some other components, that can be used to display a list of elements. These components are not compatible with the data source toolbar, though.

Select Component

The SelectComponent can be used to show a list of entries using an auto complete control. It is defined in the SelectModule.

Ordered List Component

The OrderedListComponent can be used to display a simple list of <Name, Value> entries. It's defined in the OrderedListModule

4. Other Components

Here are some components, that are connected to listings, but do not really fit into the other categories.

4.1 Selected Elements

Because all view component can contain a multi select feature, it is possible to select some items. If you like to check, which elements are selected across pages, you can use a SelectedElementsComponent that shows, how many elements are selected. This component is clickable, which opens a side sheet with all elements inside a table.

4.2 Foreign Key Picker

There are two foreign key picker dialogs available.

4.2.1 Normal Picker

The classic data picker is defined inside the FkAdvancedPickerComponent. It contains a FkSelectorComponent, that could be used on other components as well.

4.2.2 Hierarchical Picker

The hierarchical picker is defined in the FkHierarchicalDialogComponent and displays the data inside a searchable tree.
