Identity Management

In this section, there will be short description of the main components, that are important for the identity management process.

1. Data Explorer 'Identities' section

The identities section of the data explorer is defined in the DataExplorerIdentitiesComponent and can be used in two different contexts:

  1. as an admin
  2. as a manager

The manager view is filtered by identities, that are reporting to the user, while the admin view shows all identities in the system.

Each identity opens a side sheet, that displays more information about the identity.

This library defines the following sub components, that are part of this side sheet:

  • a tab control with the following sub components:

    • ObjectHyperviewComponent: displays a hyper view for the identity.
    • OrgChartComponent: displays an organizational chart for the identity.
    • ObjectHistoryComponent: displays the history of the identity object. This is defined in QBM.

  • AssignmentsComponent: A component, that shows the memberships of the selected entity. The component looks quite similar to the data explorer control. The registration of membership tabs is quite similar, too.

  • in addition to that, it is possible to register other tabs using the ExtService, which is part of the QBM library.

2. Address Book

The AddressbookComponent lists all identities, that are part of the company aka all identities, that are part of the Person database table. It is an read only view, that opens a read only side sheet as well.

3. Profile

The ProfileComponent contains the page, that is displaying different user information. The user is able to edit his main data, maintain his mail subscriptions (MailSubscriptionsComponent) and manage their password information like their security keys (SecurityKeysComponent) and personal password questions (PasswordQueryComponent).

Other tabs can be linked to this pages as well, by register them to the ExtService, which is part of the QBM library.

4. User Model

The user model is defined inside the UserModelService. It contains functions, that get user configurations, like activated features, pending items or direct reports.
