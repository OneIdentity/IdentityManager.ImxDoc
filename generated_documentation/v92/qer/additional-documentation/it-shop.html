IT Shop

This contains all components, that are needed for a complete request workflow.

1. New Request

The starting point of the request life cycle is defined inside the NewRequestComponent. The content of this page has several tabs, each containing a special component.

  1. NewRequestProductComponent: The main page, where the user is able to select products ordered by service categories.
  2. NewRequestPeerGroupComponent: In this tab, the user can select products, that other peers of his has requested as well.
  3. NewRequestReferenceUserComponent: In this tab, it is possible, to select products, an other specific identity has requested as well.
  4. NewRequestProductBundleComponent: Here it is possible, to request multiple products, that are part of a product bundle.

Each component has a set of other sub components, that will not be described in this overview, though.

2. Parameter Editor

Some products, that can be requested, are equipped with request parameter. These are handled using the ParameterDataService, which converts the data given by the server to editable parameter columns, that can be edited using the CartItemEditComponent.

3. Shopping Cart

The entry point into the shopping cart is defined inside the ShoppingCartComponent. It contains a table, that lists the cart items, that are currently part of the shopping cart. The items can be edited, deleted and checked for validity. All sub components and services are part of the ShoppingCartModule. Items in the cart can be moved to an instance of ShoppingCartForLaterComponent and vice versa.

4. Pending Requests

The next step in the workflow is the approval of items. The entry point for this is the ApprovalsComponent, which lists all the requests, the user can approve / deny.

This also contains a sub component, in which inquiries for the user are listed. (see: InquiriesComponent).

All the sub components are part of the ApprovalsModule, which includes the components on the corresponding side sheet.

5. Request History

All request, that have been made in a decent amount of time are listed in the RequestHistoryComponent. There you can filter the entries, look at their details an do some actions as well. Those actions are part of the RequestActionService and would be described there.

6. Archived Requests

Requests, that are quite old, will be archived and can be viewed using the ArchivedRequestsComponent. Items in this list can only be viewed, with no further actions available.

7. Editors for Items

Using the IT shop, there are some editors for adding new service categories, service items and even the IT shop pattern.

7.1. IT Shop Pattern

IT shop pattern can be edited using the ItshopPatternComponent. It is defined inside the ItshopPatternModule, together with the sub components, that can be used in this context.

7.2. Service Categories

For editing service categories, the ServiceCategoriesModule is needed. It contains the ServiceCategoriesComponent, that displays an DataTreeWrapperComponent, representing the service category structure of the IT shop. It is possible to add new categories, remove empty ones and edit their information.

7.3. Service Items

The service item functionality is part of the ServiceItemsEditModule. The entry point for this component is the 'ServiceItemsEditComponent', that lists all the service items available. Clicking on an entry opens a ServiceItemsEditSidesheetComponent, where the information on the service item can be edited.

7.4. Workflow Editor

All components, that are used by this editor are part of the ApprovalWorkFlowModule. The entry point for development is the ApprovalWorkflowHomeComponent which contains a list of the available workflows.
